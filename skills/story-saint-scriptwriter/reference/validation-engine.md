# Validation Engine: Algorithmic Tests for Every Level

This reference contains every test, check, and gate derived from Mamet's method, organized by the level they apply to. Run them in order. A failure at any gate means: go back to the previous level and fix the structure — never fix by adding narration, decoration, or explanation.

## 1. Throughline Validation

Run these on the superobjective BEFORE proceeding to beats.

### T1: Active Verb Test
- PASS: The superobjective uses an active, transitive verb → "to get," "to sell," "to rid," "to win," "to steal," "to escape"
- FAIL: Uses a stative/passive verb → "to be," "to feel," "to understand," "to realize," "to become," "to learn"
- FIX: Replace the stative verb with the concrete action the protagonist must perform. "To understand loss" → "To recover the stolen heirloom." "To find peace" → "To bury the dead."

### T2: Concrete Object Test
- PASS: The objective contains a concrete noun or outcome you could photograph being achieved → "a retraction," "the pig," "the stolen documents," "the escape route"
- FAIL: The objective contains an abstract noun → "respect," "happiness," "truth," "redemption," "meaning"
- FIX: Ask "what is the PHYSICAL SIGN that this abstract thing has been achieved?" Use that sign as the objective. "Respect" → "a retraction" (the stamp on the book). "Redemption" → "the pig gives the girl a ride" (the visible proof that danger is gone).

### T3: Binary Cap Test
- PASS: You can name the exact moment the story is over → "when the grade is stamped" / "when the farmer walks home with two pigs" / "when the retraction is categorically denied"
- FAIL: You cannot name a definitive endpoint → the objective trails off, could go on indefinitely
- FIX: Add a physical cap. "To gain respect" has no cap. "To get a retraction" caps at the stamp. "To sell the pig" caps at the transaction or final refusal.

### T4: External Trigger Test
- PASS: The story starts because something HAPPENED TO the protagonist from outside → the plague hits Thebes, the pig attacks the child, Charles Foster Kane dies
- FAIL: The story starts because the protagonist decided/felt/realized something → "one day he woke up and decided..."
- FIX: Find the external event that FORCED the decision. The pig didn't attack because the farmer felt like selling — the attack forces the sale.

### T5: Word Count Test
- PASS: Superobjective stated in ≤12 words → "A farmer must rid his house of a dangerous pig"
- FAIL: Requires more than 12 words
- FIX: You're bundling multiple objectives or adding context. Strip to one verb + one object.

### T6: Single Reason Test
- PASS: The protagonist has exactly ONE reason for pursuing the objective
- FAIL: The protagonist has two or more reasons → "the pig is dangerous AND the family is broke"
- FIX: Pick the stronger reason. Cut the other. Two reasons = split focus = no focus. "I was late because the bus strike AND my aunt fell downstairs" convinces nobody. Pick one.

**Gate: The throughline must pass T1-T6 to proceed to Phase 2.**

## 2. Beat Validation

Run these on EACH beat individually, then on the beat LIST as a whole.

### Per-Beat Tests

#### B1: Verb Phrase Test
- PASS: Beat is named "to [active verb] [object]" → "to arrive early," "to prepare," "to pay homage," "to capitalize on a golden opportunity"
- FAIL: Beat is named as a noun, adjective, or result → "nervousness," "the locked door," "failed attempt," "judgment"
- FIX: Convert to the protagonist's active step. "Failed attempt" → "to capitalize on a golden opportunity" (the attempt itself, not its result). "Judgment" → "to receive judgment" (if protagonist-driven) or restructure.

#### B2: Removal Test
- PROCEDURE: State the throughline. List all beats. Remove this beat. Tell the story without it.
- PASS: The story becomes incomprehensible or a logical step is missing → the beat is essential
- FAIL: The story still works without it → the beat is ornamental
- FIX: Cut the beat. If you feel attachment to it, that attachment is the conscious mind wanting to be "interesting." Cut it anyway.

#### B3: Superobjective Service Test
- PASS: The beat is a STEP TOWARD the superobjective → "arriving early" serves "getting a retraction"
- FAIL: The beat serves "the character," "the setting," "the mood," or "the audience's understanding" rather than the superobjective → "establishing the classroom," "showing his personality," "creating tension"
- FIX: Ask "does this beat help the protagonist GET the objective?" If no → cut. Character, setting, and mood emerge from structure, not from dedicated beats.

### Beat-List Tests

#### B4: Circularity Check
- PROCEDURE: For each pair of beats (B_i, B_j where j > i), ask: "Could these be described as the same action in a different context?"
- PASS: No two beats describe the same action → each stair is climbed only once
- FAIL: Two beats are the same action recycled → "to prepare" and "to review preparation"; "to ask" and "to plead"
- FIX: Merge into one beat or cut the weaker one. If the knife cuts bread in verse 1, it must not cut cheese in verse 2.

#### B5: Escalation Check (Leadbelly's Knife)
- PROCEDURE: For each consecutive pair (B_i, B_i+1), ask: "Are the stakes higher, the commitment deeper, or the path narrower in B_i+1?"
- PASS: Each beat escalates → first verse knife cuts bread, second verse shaves, third verse kills
- FAIL: Stakes are flat or decrease → the story plateaus
- FIX: Reorder beats so stakes increase. If reordering doesn't work, a beat is probably redundant.

#### B6: Abstraction Level Check
- PROCEDURE: Rate each beat's specificity on a rough scale. "To travel to town" is scene-level. "To blink" is micro-action-level. "To seek meaning" is thematic-level.
- PASS: All beats sit at the same level → all are scene-level steps
- FAIL: Beats are at mixed levels → some are zoomed way in, others zoomed way out
- FIX: Normalize. Split too-large beats into scene-level steps. Merge too-small beats into their parent beat.

#### B7: Completeness Check
- PASS: The final beat directly produces the superobjective OR results in its irrevocable denial
- FAIL: The final beat is a step that requires additional unstated steps before the objective resolves
- FIX: Add the missing beats, or restructure so the final beat IS the resolution.

#### B8: Count Check
- PASS: 3-8 beats total
- FAIL: <3 beats → the story may be too simple or beats are too coarse → split further
- FAIL: >8 beats → you likely have sub-beats mixed with beats, or a multi-scene structure → group into scenes first, then beat each scene

**Gate: All beats must pass B1-B3 individually, and the list must pass B4-B8 as a whole, to proceed to Phase 3.**

## 3. Shot Validation

Run these on EACH shot individually, then on each beat's shot SEQUENCE.

### Per-Shot Tests

#### S1: Word Count Test
- PASS: Shot description is ≤10 words → "A man walks down an empty hallway"
- FAIL: >10 words → description is overloaded
- FIX: Split into two shots or strip adjectives.

#### S2: Inflection Scan
Scan the shot description for ANY of these categories. If found → FAIL.

**Emotional adjectives:** nervously, anxiously, reverently, humbly, determinedly, angrily, sadly, desperately, hopefully, lovingly, fearfully, contrite, respectful, ominous
**Narrative context words:** because, after, since, knowing, realizing, in order to, so that, as if, trying to
**Interiority words:** thinking, feeling, remembering, hoping, wondering, deciding, noticing
**Result words:** defeated, victorious, relieved, satisfied, frustrated (these describe the result, not the image)
**Dilution words:** just, kind of, sort of, maybe, somewhat, a little bit

- PASS: Zero hits across all categories
- FAIL: Any hit → the shot is inflected
- FIX: Remove the flagged word. The shot should describe ONLY what a camera sees. "A man nervously walks" → "A man walks quickly." "She sits, defeated" → "She sits." The structure (juxtaposition) carries the emotion, not the shot.

#### S3: Photographability Test
- PASS: A single camera in a fixed or simple-motion setup can capture exactly this image → "hand turns a doorknob," "feet walk down a corridor," "an empty desk"
- FAIL: Requires time-lapse, dissolve, split-screen, VFX, montage-within-shot, or narration to make sense → "thirty hours pass," "he remembers his childhood," "the city transforms over decades"
- FIX: If it can't be a photograph, it's not a shot. Either it's narration (cut it) or it needs to be decomposed into multiple shots whose juxtaposition implies the concept.

#### S4: Autonomy Test
- PASS: The shot does NOT try to carry the meaning of the whole scene → a doorknob is a doorknob, not a symbol of the entire pursuit of respect
- FAIL: The shot is "loaded" with whole-scene meaning → a "retraction notebook," a "contrite expression," a "determined stride"
- FIX: The nail does not need to look like the house. Make each shot do only its job. The cut does the synthesis.

### Shot-Sequence Tests (per beat)

#### S5: Juxtaposition Test
- PROCEDURE: For each consecutive pair of shots within a beat, ask: "Does Shot A placed next to Shot B produce the idea of [the beat's one-word concept]?"
- PASS: The idea emerges from the CUT between shots, not from either shot alone
- FAIL: The idea is in one shot (the other is redundant) OR the juxtaposition produces a different/wrong idea
- FIX: Replace one or both shots so that meaning emerges from their pairing. Reference: birds snapping twig + fawn raising head = alertness. Neither shot alone = alertness.

#### S6: Minimum Shot Count
- PASS: Each beat has ≥2 shots (juxtaposition requires at least two images)
- FAIL: A beat has only 1 shot → there's no cut, therefore no juxtaposition, therefore no meaning creation
- FIX: Add a second shot that, paired with the first, creates the beat's idea through juxtaposition.

#### S7: Maximum Shot Count
- PASS: Each beat has ≤5 shots
- FAIL: >5 shots → the beat may actually be two beats, or shots are redundant
- FIX: Check if the beat should be split. If not, apply the Removal Test to each shot — cut any that aren't essential to the juxtaposition chain.

#### S8: Information Withholding Check
- PROCEDURE: For each shot, ask: "Am I SHOWING something the audience could INFER from juxtaposition alone?"
- PASS: The shot adds new visual information the audience needs
- FAIL: The shot explains or confirms what the audience already inferred → it's redundant exposition
- FIX: Cut the shot. Trust the audience. Example: if you show mom grabbing a broom and running out, you do NOT also need to show the pig attacking. The cut implies the danger.

**Gate: All shots must pass S1-S4 individually, and each beat's sequence must pass S5-S8, to proceed to Phase 4.**

## 4. Structure Validation (Full Assembly)

Run these on the COMPLETE shot list, reading all beats in sequence.

### V1: Syllogism Test
- STATE THE SYLLOGISM: "If [disordering event], then [protagonist pursues objective through beats 1-N], until [order is restored at final beat]."
- PASS: This reads as a single coherent sentence
- FAIL: The sentence is incoherent, has logical gaps, or requires "and then also..." connectors
- FIX: The logical gap indicates a missing beat. Go back to Phase 2.

### V2: Silent Movie Test
- PROCEDURE: Remove ALL dialogue from the shot list. Read only the image descriptions in sequence.
- PASS: The story is clear from images alone → dialogue is sprinkles on the ice cream
- FAIL: The story is unclear without dialogue → the shots aren't carrying the narrative
- FIX: The structure is wrong, not the dialogue. Go back to Phase 3 and redesign shots so juxtaposition carries the story. Never fix by adding dialogue.

### V3: Burn-the-First-Reel Test
- PROCEDURE: Delete the first beat entirely. Read beats 2-N.
- PASS: The story BREAKS without beat 1 → beat 1 is essential → keep it
- FAIL: The story still works → beat 1 is unnecessary → your real story starts at beat 2
- ITERATE: Repeat with the new first beat until cutting a beat breaks the story. The surviving first beat is your true beginning. "Get into the scene late."

### V4: Surprise + Inevitability Test
- PROCEDURE: Read only beat 1 and the final beat. Then read ALL beats.
- PASS: (a) The final beat is NOT predictable from beat 1 alone (surprise), AND (b) Given ALL beats in sequence, the final beat is the only logical conclusion (inevitability). BOTH must be true.
- FAIL: The ending is predictable from the start (no surprise) → restructure middle beats to create genuine turns. OR the ending feels arbitrary (no inevitability) → the beat sequence has logical gaps.

### V5: Therefore/But Test
- PROCEDURE: Between each consecutive beat, insert either "therefore" or "but." Read the sequence.
- PASS: Every transition is "therefore" (logical consequence) or "but" (complication) → "He arrives early, THEREFORE he prepares, THEREFORE he pays homage, BUT the opportunity is lost..."
- FAIL: Any transition requires "and then" (mere sequence without causation) → the beats are episodic, not dramatic
- FIX: Restructure beats so each flows causally from the previous. If a beat doesn't follow from the prior beat by "therefore" or "but," it's either out of order or unnecessary.

### V6: Information Debt Audit
- PROCEDURE: At each shot, list what the audience KNOWS vs. what they WANT TO KNOW.
- PASS: At every point, the audience's want-to-know list is non-empty → they are pulled forward by curiosity
- FAIL: At any point, the audience knows everything they need to know → there's no engine pulling them forward
- FIX: Withhold more. If you're showing too much too early, restructure so information is revealed through juxtaposition at the latest possible moment.

**Gate: The structure must pass V1-V6 to proceed to output generation.**

## 5. Prop/Set/Continuity Validation

### P1: Throughline Prop Test
- For every prop: "Does this prop serve the throughline?"
- RIGHT ANSWER: "The notebook must be recognizable" (serves the throughline of presenting the case)
- WRONG ANSWER: "The notebook should look academic/contrite/professional" (serves characterization, not throughline)

### P2: Recognition Chain Test
- If a prop appears in Beat X and must connect to Beat Y, flag it: MUST_MATCH.
- The prop's essential quality is RECOGNIZABILITY, not expressiveness.
- Example: the handkerchief on the pig must be visible in both the "golden opportunity" beat and the final "reward" beat.

### P3: Sign Test
- If you find yourself wanting a sign, label, or text on screen to convey information → the structure is failing. Go back to the shots.
- Signs don't work because the audience won't read them — they're watching for what happens NEXT.

## 6. Dialogue Validation

### D1: Want Test
- PASS: Every line of dialogue is the character speaking to GET WHAT THEY WANT → "Where the hell were you yesterday?"
- FAIL: The line conveys information TO THE AUDIENCE → "Hello Jack, I'm coming over because I need the money you borrowed"
- FIX: Cut the exposition. The shots carry the information. Dialogue is only for the character's pursuit.

### D2: Removal Test
- PROCEDURE: Remove the line. Does the story still work from the shots alone?
- PASS: Yes → the dialogue is a gloss, which is correct
- FAIL: No → the dialogue is load-bearing, which means the shots are failing. Fix the shots, not the dialogue.

### D3: Subtitling Test
- If the film were dubbed into a language the viewer doesn't speak, would the story still track?
- PASS: Yes → the shots carry the story
- FAIL: No → restructure shots

## 7. Failure Recovery Rules

When a test fails, the fix is ALWAYS structural, NEVER decorative.

**Never fix by:**
- Adding narration (voice-over, text on screen, character exposition)
- Adding "establishing shots" (wide shots of buildings, signs, locations)
- Adding emotional inflection to shots ("make it look more ominous")
- Adding dialogue to explain what the shots don't convey
- Adding ADR/looping (dubbed explanatory dialogue)
- Making the actor/AI perform the emotion (the structure performs the emotion through juxtaposition)

**Always fix by:**
- Going back one level (shot fails → re-examine the beat; beat fails → re-examine the throughline)
- Simplifying (Occam: between two valid options, pick the simpler one)
- Cutting (if a shot/beat isn't essential, removing it is always safer than keeping it)
- Restructuring juxtaposition (change WHICH shots are next to each other, not what's IN the shots)

The throughline is the compass. When lost, return to: "What does the protagonist want? What is THIS beat's step toward it? What two images juxtaposed create THIS idea?" These three questions solve 90% of problems.
