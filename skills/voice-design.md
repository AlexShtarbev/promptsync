# Voice Design

Designs production-ready character voices for ElevenLabs synthesis. Takes character descriptions and produces: a voice profile for each character, ElevenLabs voice selection criteria, per-emotional-state settings, and voice direction language.

**This skill owns voice design and ElevenLabs configuration. animation-prompts owns prompt generation and uses the voice profiles produced here when writing `elevenlabs.md`.**

---

## Step 1 — Voice Profile Framework

Before touching ElevenLabs, design each character's voice on paper. A voice profile answers: *what does this person sound like, and why?*

### Profile Template

```
## [Character Name]

Role: [protagonist / antagonist / narrator / supporting / background]
Archetype: [e.g. fallen angel, weary elder, zealous soldier, grieving mother]

--- PHYSICAL BASIS ---
Age range: [e.g. 40s–50s in appearance]
Build: [affects resonance — lean/light vs broad/heavy]
Energy level: [low/slow vs high/fast baseline]

--- VOCAL QUALITIES ---
Register: [bass / baritone / tenor / alto / mezzo / soprano]
Texture: [smooth / gravelly / breathy / crisp / nasal / resonant]
Weight: [light / medium / heavy — how much space the voice occupies]
Pace: [deliberate / measured / quick / variable]
Accent/dialect: [neutral / RP British / American Southern / Middle Eastern / etc.]

--- SPEECH PATTERNS ---
Sentence structure: [long complex sentences / short declarative / fragments / poetic]
Vocabulary register: [archaic / elevated / plain / modern / mixed]
Verbal tics: [pauses before key words / trailing sentences / rhetorical questions / none]
Volume dynamic: [consistently quiet / consistently loud / wide range / narrow range]

--- EMOTIONAL BASELINE ---
Default affect: [what emotion the character carries when saying nothing dramatic]
Emotional range: [narrow — controlled at all times / wide — expressive and variable]
Suppression style: [how they sound when holding something back]

--- DISTINGUISHING QUALITY ---
[One sentence: the single most recognisable thing about this voice.
 Example: "A quiet authority that never rises — the voice that doesn't need to be loud to fill a room."]
```

### Worked Example

```
## LUCIFER

Role: antagonist (or fallen protagonist)
Archetype: exiled angel — beautiful, cold, dangerously composed

--- PHYSICAL BASIS ---
Age range: ageless, but carries the weight of aeons — sounds mid-40s
Build: lean, precise — no wasted breath
Energy level: low baseline, can accelerate sharply

--- VOCAL QUALITIES ---
Register: deep baritone
Texture: smooth with a faint edge — silk over steel
Weight: heavy — commands space without effort
Pace: deliberate; treats silence as punctuation
Accent/dialect: neutral with elevated diction — no region, no era

--- SPEECH PATTERNS ---
Sentence structure: long, structurally perfect sentences; knows grammar as power
Vocabulary register: elevated, slightly archaic — "what you have done" not "what you did"
Verbal tics: pauses before the final word of a sentence, letting it land alone
Volume dynamic: narrow — rarely loud; intensity comes from stillness, not volume

--- EMOTIONAL BASELINE ---
Default affect: composed grief — beautiful sadness underneath absolute control
Emotional range: narrow — even at extremes, the control barely breaks
Suppression style: slower pace, longer pauses, slightly lower register

--- DISTINGUISHING QUALITY ---
"The voice that makes you lean in — quieter when angry than when calm."
```

## Step 2 — ElevenLabs Model & Synthesis Engine

### Recommended Model

Use `eleven_v3` for all new productions. It has the best emotional range, accent fidelity, and nuance control of any ElevenLabs model.

Fallback: `eleven_turbo_v2_5` if generation speed is a hard constraint — expect a meaningful quality drop on emotionally subtle lines.

### How v3 Controls Delivery (No SSML)

v3 does **not** support SSML. Delivery is controlled through two mechanisms:

**1. Audio tags** — prepend directional cues to the text:
```
[whispers] …wronged you?
[quietly resolute] I am ready!
[barely audible, breaking] God — have I…
[softly, searching] What is life…
```
Match the tag to the voice's actual character — a tag like `[shouts]` on a voice trained for intimacy will produce inconsistent results.

**2. Punctuation** — ellipses for pauses, em dashes for breaks, capitalization for emphasis:
```
God — have I…        ← dash = internal break, ellipsis = trailing off
Music is my LIFE.    ← caps = weighted emphasis
```

Do not use SSML `<break>`, `<prosody>`, or `<speak>` tags — they are not parsed by v3 and will appear as literal text in the output.

### Single-Take with CapCut Splitting

When generating a full VO as one audio file for splitting in a video editor:
- Use audio tags + line breaks between fragments to create natural pause gaps
- Use `[long pause]` between fragments where a long silent shot exists
- Gaps will vary per generation — use the waveform view in the editor to find cut points
- Do not rely on precise gap timing; adjust in the editor after generation

## Step 3 — ElevenLabs Voice Selection

### Library Voice vs Voice Clone

| Situation | Recommendation |
|-----------|---------------|
| Character is fictional, no specific real-world reference | Use ElevenLabs library — audition 3–5 candidates |
| Character is based on a real person (historical, celebrity) | Clone if you have clean audio; otherwise find closest library match |
| Character needs extreme consistency across 100+ lines | Clone — library voices can drift subtly across sessions |
| Character has only 5–10 lines total | Library voice is fine |
| Character needs a very specific accent or texture the library doesn't have | Clone from sourced audio |

### Voice Design Feature (Custom Accent / Character)

When the library doesn't have the right accent or character texture, use ElevenLabs **Voice Design** to generate a custom voice from a text description:

```
[Example — Viennese-accented narrator]
Austrian-accented English, male, late fifties, baritone register, voice slightly roughened by age,
intimate and measured, reads as if speaking to himself not an audience, European formal diction
with Viennese consonant weight, not theatrical, not an announcer, a philosopher's voice —
gravitas without performance, warmth held under restraint
```

Generate 3–4 variations. Screen each with the audition lines below. Voice Design is faster than sourcing clone audio and produces consistent results for fictional or historical characters where no real-world audio exists.

### How to Audition Library Voices

Test every candidate voice with these three lines — they cover register, pace control, and emotional range:

```
Test line 1 (neutral, declarative):
"The city was silent. No wind. No birds. Only the sound of what had just ended."

Test line 2 (controlled intensity, elevated register):
"You knew what this would cost. You chose it anyway."

Test line 3 (suppressed grief or quiet fury — pick what fits the character):
"I do not blame you. That is the worst part."
```

**What to listen for:**
- Does the register (bass/baritone/etc.) match the profile?
- Does the texture (smooth/gravelly/breathy) match?
- Does the pace feel natural at the character's intended speed?
- Does emotional subtext come through in test line 3 without over-performing?
- Does it sound distinct from other characters already cast?

**Eliminate immediately if:**
- The voice sounds modern/casual for a period or elevated character
- There are audible artefacts or instability on longer phrases
- It sounds like any other character already in the project

### Voice Clone Source Requirements

If cloning from external audio:
- Minimum 3 minutes of clean, single-speaker audio
- No background music, reverb, or heavy processing
- Multiple emotional registers preferred (not all the same tone)
- Consistent microphone/recording quality throughout

## Step 4 — Per-Character Settings

### Settings Template (per character)

```
## [Character Name] — ElevenLabs Settings

Voice ID: [library ID or "CLONED — [source description]"]

DEFAULT (neutral/baseline scene — tune to produce the character's Default affect from the voice profile):
  stability:        [value]
  similarity_boost: [value]
  style:            [value]
  speed:            [value]

EMOTIONAL STATES:
  [state_name]:
    stability:        [value]
    similarity_boost: [value]
    style:            [value]
    speed:            [value]
    notes:            [what this state sounds like, when to use it]
```

### Settings Reference

| Parameter | What it controls | Effect of increasing |
|-----------|-----------------|---------------------|
| `stability` | Consistency of delivery | More uniform, less expressive — higher = flatter |
| `similarity_boost` | How closely it stays to the voice clone/model | Higher = less drift, more locked to the source voice |
| `style` | Expressiveness / performance intensity | Higher = more emotional colouring, more variable |
| `speed` | Speaking rate | Higher = faster; 1.0 is neutral |

### Emotional State Presets

Use these as starting points — adjust per character based on their profile's emotional range.

```
COMMANDED / AUTHORITATIVE (`commanded_authoritative`):
  stability: 0.65    — controlled, no wobble
  similarity_boost: 0.85
  style: 0.35        — presence without over-performance
  speed: 0.90        — slightly slower; authority doesn't rush

GRIEF / LOSS (`grief_loss`):
  stability: 0.40    — some fragility is correct
  similarity_boost: 0.85
  style: 0.65        — higher expression, but not melodramatic
  speed: 0.82        — slower; grief weighs words down

SUPPRESSED FURY (`suppressed_fury`, anger held back):
  stability: 0.70    — the control IS the fury
  similarity_boost: 0.85
  style: 0.45        — contained; the danger is in the stillness
  speed: 0.88        — deliberate, each word placed

OPEN FURY / OUTBURST (`open_fury`):
  stability: 0.30    — allow instability; it sounds raw
  similarity_boost: 0.80
  style: 0.80        — high expression
  speed: 1.15        — urgency drives the pace

TENDER / INTIMATE (`tender`):
  stability: 0.50
  similarity_boost: 0.85
  style: 0.55
  speed: 0.88        — soft pace; not rushed

FEARFUL / DESPERATE (`fearful_desperate`):
  stability: 0.35    — instability reads as fear
  similarity_boost: 0.82
  style: 0.70
  speed: 1.10        — fear accelerates speech

NARRATION (`narration`, VO, no scene emotion):
  stability: 0.72    — consistent across all scenes
  similarity_boost: 0.90  — lock the narrator voice
  style: 0.25        — measured, not performative
  speed: 0.95        — slightly slower; narration lands better with space
```

### Worked Example — Lucifer Settings

```
## LUCIFER — ElevenLabs Settings

Voice ID: [TBD — target: deep baritone, smooth, neutral accent]

DEFAULT (composed, baseline):
  stability:        0.62

EMOTIONAL STATES:

  suppressed_fury:
    stability:        0.72

```

## Step 5 — Voice Direction Language

Voice direction in `elevenlabs.md` operates at two levels in v3:

**1. Audio tags (in the prompt text) — these DO influence synthesis:**
Prepend directional cues directly to the spoken line. v3 reads these as delivery instructions.
```
[whispers] I have wronged you.
[quietly resolute] I am ready.
[barely audible, breaking] God — have I…
```

**2. Direction notes (after the line) — for human review only:**
These are not parsed by the TTS engine. They help you judge the take and guide regeneration passes.

### What Works

Direction that describes **subtext and physical state** affects delivery far more reliably than direction that names emotions directly.

| Instead of... | Write... |
|---------------|---------|
| "Say this sadly" | "The decision was made long ago. This is just the saying of it." |
| "Sound angry" | "Jaw tight. Each word placed like a stone." |
| "Be commanding" | "Not asking. Not threatening. Stating. The room already knows." |
| "Sound afraid" | "The breath is shorter than it should be. The words come faster than intended." |
| "Be emotional" | "Holding something back that is trying very hard to come out." |

### Direction Note Format

```
[SHOT shot_id] [STATE state_name]
"Verbatim line of dialogue."
// [One sentence of subtext/physical state direction.]
// [Optional: one sentence on what NOT to do if there's a specific failure mode to avoid.]
```

Example:
```
[SHOT shot_07] [STATE suppressed_fury]
"You knew what this would cost. You chose it anyway."
// The anger is so old it has become indistinguishable from grief. No heat in the voice — the coldness is the accusation.
// Avoid: raising volume on "anyway" — the line lands harder if it drops.
```

### Direction Anti-Patterns

**Do not write:**
- Adverbs describing how to say the line: "Say this slowly and sadly" — useless
- Emotional labels without physical grounding: "This is the saddest moment" — not actionable
- Contradictory directions: "Angry but calm and also hopeful" — model cannot resolve this
- Length over 2 sentences — longer direction loses coherence in synthesis

## Step 6 — Voice Consistency Rules

### Across Scenes

- Always use the **same Voice ID** for a character throughout the project — never swap
- Always use the **same `similarity_boost`** value for a character — this is the identity anchor
- Allow `stability`, `style`, and `speed` to shift per emotional state — that's the performance
- Regenerate from the same voice ID if a take needs redoing — do not substitute a different voice and "match" later in post; the timbre will be wrong

### Cross-Character Separation

Each character must be distinct on **at least two axes** from every other character:

| Axes | Examples |
|------|---------|
| Register | bass vs tenor; contralto vs soprano |
| Texture | smooth vs gravelly; crisp vs breathy |
| Pace | deliberate vs quick |
| Weight | heavy/occupying vs light/urgent |
| Accent | neutral vs regional; archaic vs modern |

**If two characters are too similar:** adjust one character's settings or re-cast. Similar voices in the same scene create confusion even when the dialogue is clear.

### Narrator vs Characters

The narrator's voice should be **outside** the world — not cast in the same register or texture cluster as any main character. It occupies a different sonic layer. If the narrator IS a character (unreliable narrator, character retelling their own story), the voice ID can be the same but the NARRATION settings preset should be used, not the character's emotional presets.

## Storyboard Emotion → Voice State Mapping

The storyboard's Emotion column drives camera language (`cinematography.md`) and color/palette (`nanobanana-artistry.md`) through formal lookup tables. This table provides the equivalent mapping for voice states, ensuring visual and audio emotion are aligned by default.

Use this table as a **default**. Override when the dialogue subtext diverges from the shot's visual emotion (e.g., a character says something tender in a scene staged as tension — the voice follows the dialogue, not the camera).

| Storyboard Emotion | Default Voice State | Rationale |
|---|---|---|
| Power / Dominance | `commanded_authoritative` | Control, no wobble, slower pace |
| Vulnerability / Defeat | `grief_loss` or `tender` | Fragility, lower energy |
| Heroism / Defiance | `commanded_authoritative` | Presence without heat |
| Isolation / Loneliness | `grief_loss` | Carried weight, not fresh |
| Intimacy / Tenderness | `tender` | Soft pace, warmth |
| Tension / Dread | `suppressed_fury` | Control IS the danger |
| Disorientation / Madness | `open_fury` with stability 0.30 | Instability reads as fracture — use sparingly |
| Awe / Wonder | `tender` or default | Quiet openness |
| Hope / Aspiration | default with speed 0.92+ | Slight lift, not performative |
| Loss / Grief | `grief_loss` | Old grief, not fresh sobbing |
| Determination / Resolve | `commanded_authoritative` | Stripped, direct |
| Urgency / Chase | `fearful_desperate` or `open_fury` | Fear accelerates; fury drives |
| Fear | `fearful_desperate` | Instability, faster pace |
| Contempt / Cruelty | `suppressed_fury` | Cold, deliberate |
| Anger / Rage | `open_fury` (if erupting) or `suppressed_fury` (if contained) | Match the containment level |
| Shame / Humiliation | `grief_loss` | Turned inward, low energy |
| Contemplation | default or `narration` | Measured, not performative |
| Liberation / Release | default with style 0.50+ | Slight lift in expression |
| Nostalgia / Memory | default with speed 0.88 | Warm, slightly slower, reflective |
| Revelation / Surprise | default with stability 0.40 | Brief instability, then settling |
| Emptiness / Numbness | `narration` with style 0.15 | Flat, affect-less, stripped |
| Anticipation / Buildup | `suppressed_fury` with speed 0.92 | Contained energy, slight acceleration |
| Shock / Panic / Rupture | `fearful_desperate` with stability 0.30 | Maximum instability, breaking |
| Entrapment / Compression | `suppressed_fury` | Tightening, no escape in the voice |

**Entry format patterns** — the table uses three formats:

1. **Preset name alone** (e.g., `suppressed_fury`) — use the full preset as defined in Step 4 Emotional State Presets
2. **Preset with parameter override** (e.g., `open_fury with stability 0.30`) — start from the named preset, override the specified parameter
3. **Default with parameter override** (e.g., `default with speed 0.92+`) — start from the character's DEFAULT settings, override the specified parameter

When an entry uses format 2 or 3, the override applies on top of the base settings — all other parameters stay at the base values.

**Usage:** When `animation-prompts` selects the emotional state preset for a shot, start with this table, then check whether the dialogue itself suggests a different state. Document any override in the direction note: `// Override: storyboard emotion is "tension" but dialogue is a tender confession — using tender state.`

## Integration with animation-prompts

When `animation-prompts` generates `elevenlabs.md`, it must:

1. Pull the **Voice ID** and **default settings** from `voice_design.md`
2. Select the correct **emotional state preset** — start with the Storyboard Emotion → Voice State Mapping table above, then adjust based on the shot's dialogue subtext
3. Write direction notes using **subtext/physical state language** (not emotion labels)
4. Match the **speed setting** to the `speech_pace` field from the shot JSON:
   - `dramatic` → speed ≤ 0.90
   - `normal` → speed 0.95–1.00
   - `fast` → speed 1.10–1.20
   (Override with character's emotional state if the state has its own speed setting)

### Precedence Rule

`emotional state speed` **overrides** `speech_pace speed` when they conflict. Character voice design trumps generic pace tiers.

### Refining `speech_pace` During Voice Design

The storyboard assigns `speech_pace` as a first-pass estimate based on scene emotion (storyboard.md). During voice design, review each dialogue shot's `speech_pace` against the character's vocal profile and the emotional state presets. If a shot's pace doesn't match the character's voice design (e.g., a "dramatic" pace assigned to a rapid-fire character whose natural register is fast), update the `speech_pace` in the shot's `shot.md` frontmatter and the storyboard TSV's Notes column. Document: "speech_pace updated from X to Y per voice design — [reason]."

## Quick Reference — New Project Checklist

- [ ] Voice profile written for every speaking character and narrator
- [ ] Each character has a distinguishing vocal quality (one sentence)
- [ ] No two characters share the same register + texture combination
- [ ] Narrator voice is distinct from all character voices
- [ ] ElevenLabs voice auditioned with all three test lines
- [ ] Voice ID confirmed and recorded in `voice_design.md`
- [ ] Default settings block written for every character
- [ ] At least 3 emotional state presets defined per major character (for 15s Shorts with 1-2 lines: 1-2 presets is sufficient — scale to project scope)
- [ ] Direction language uses subtext/physical state, not emotion labels
- [ ] `speech_pace` assigned for every dialogue shot in the storyboard (`dramatic | normal | fast` — not left as `null`)
- [ ] `voice_design.md` is complete before any `elevenlabs.md` is generated
